= SnapMirror Cloud backup su storage a oggetti
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
_SnapMirror Cloud_ è una tecnologia di backup e ripristino progettata per gli utenti ONTAP che vogliono migrare i propri flussi di lavoro di protezione dei dati verso una soluzione di storage a oggetti cloud o on-premises. Le organizzazioni che abbandonano le architetture di backup su nastro tradizionali possono utilizzare lo storage a oggetti come repository alternativo per la conservazione e l'archiviazione dei dati a lungo termine. SnapMirror Cloud offre la replica da ONTAP a storage a oggetti come parte di una strategia di backup incrementale permanente.

SnapMirror Cloud è una funzionalità ONTAP con licenza. SnapMirror Cloud è stata introdotta in ONTAP 9.8 come estensione della famiglia di tecnologie di replica SnapMirror. Mentre SnapMirror viene spesso utilizzato per i backup da ONTAP a ONTAP, SnapMirror Cloud utilizza lo stesso motore di replica per trasferire snapshot dai sistemi ONTAP ai backup su storage a oggetti compatibili con S3.

Pensato per i casi d'uso di backup, SnapMirror Cloud supporta sia la conservazione a lungo termine che i flussi di lavoro di archiviazione. Come per SnapMirror, il backup iniziale SnapMirror Cloud esegue un trasferimento baseline di un volume. Per i backup successivi, SnapMirror Cloud genera un'istantanea del volume di origine e trasferisce l'istantanea, con solo i blocchi di dati modificati, a una destinazione di storage a oggetti.

SnapMirror Cloud può essere configurato tra sistemi ONTAP e specifici storage a oggetti on-premise e cloud pubblico, inclusi Amazon S3, Google Cloud Storage e Microsoft Azure Blob Storage. Ulteriori storage a oggetti on-premise includono StorageGRID e ONTAP S3.

Oltre a utilizzare ONTAP System Manager per gestire le configurazioni SnapMirror Cloud, sono disponibili diverse opzioni di orchestrazione per gestire i backup SnapMirror Cloud:

* Numerosi partner di backup di terze parti offrono supporto per la replica SnapMirror Cloud. I fornitori partecipanti sono disponibili su link:https://www.netapp.com/blog/new-backup-architecture-snapdiff-v3/["Blog di NetApp"^].
* NetApp Backup and Recovery per una soluzione nativa NetApp per ambienti ONTAP
* API per lo sviluppo di software personalizzato per i flussi di lavoro di data Protection o l'utilizzo di strumenti di automazione



NOTE: La replica cloud di SnapMirror gestita da System Manager supporta solo StorageGrid e ONTAP S3 come destinazione dello storage a oggetti. Altri provider non sono supportati.


NOTE: L'utilizzo delle API Cloud SnapMirror è generalmente riservato ai partner di backup. I clienti che desiderano sviluppare soluzioni basate su API devono contattare il proprio team NetApp per informazioni sui termini di licenza.
