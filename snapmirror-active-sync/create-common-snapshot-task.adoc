= Crea uno snapshot comune tra gruppi di coerenza ONTAP o SVM NAS
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Oltre alle operazioni di snapshot programmate regolarmente, puoi creare manualmente una link:../concepts/snapshot-copies-concept.html["snapshot"] comune tra i volumi nel gruppo di coerenza SnapMirror primario e i volumi nel gruppo di coerenza SnapMirror secondario in una configurazione SAN o a livello SVM per le configurazioni NAS.

.A proposito di questa attività
L'intervallo programmato per la creazione degli snapshot è di 12 ore per la policy Automated Failover e di 3 ore per la policy Automated Failover Duplex.

.Prima di iniziare
* La relazione del gruppo SnapMirror deve essere sincronizzata.


.Fasi
Puoi creare uno snapshot comune utilizzando System Manager o la ONTAP CLI.

[role="tabbed-block"]
====
.System Manager
--
. Dal cluster di destinazione, vai su *Protezione > Replica*.
. Individua l'origine della relazione, seleziona image:icon_kabob.gif["Icona delle opzioni di menu"] e scegli *Aggiorna*.
. Scegli *Secondo le policy* o *Seleziona l'istantanea* per scegliere un'istantanea esistente.
. Seleziona *Aggiorna*.


--
.CLI
--
. Dal sito di destinazione, crea un'istantanea comune:
+
`snapmirror update -destination-path <destination path>`

+
Esempio di configurazione SAN:

+
[listing]
----
destination::>snapmirror update -destination-path vs1_dst:/cg/cg_dst
----
+
Esempio di configurazione NAS:

+
[listing]
----
snapmirror update -destination-path vs_b:
----
. Monitorare l'avanzamento dell'aggiornamento:
+
`destination::>snapmirror show -fields newest-snapshot`
