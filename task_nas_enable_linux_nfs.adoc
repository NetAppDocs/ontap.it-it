= Abilitare l'archiviazione NAS per server Linux utilizzando NFS con ONTAP System Manager
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Creare o modificare le VM di storage per abilitare i server NFS per la distribuzione dei dati ai client Linux.

Abilitazione di una macchina virtuale di storage nuova o esistente per il protocollo NFS mediante questa procedura.

image:workflow_nas_enable_linux_nfs.png["Riepilogo del flusso di lavoro: 1 abilitare NFS 2 criterio di esportazione volume root aperto 3 configurare DNS 4 configurare i servizi nome 5 configurare la crittografia se necessario"]

.Prima di iniziare
Assicurarsi di aver preso nota dei dettagli di configurazione per tutti i servizi di rete, autenticazione o protezione richiesti nell'ambiente.

.Fasi
. Abilitare NFS su una VM di storage.
+
** Per le nuove VM di archiviazione: selezionare *Archiviazione > VM di archiviazione*, selezionare *Aggiungi*, immettere un nome per la VM di archiviazione e nella scheda *SMB/CIFS, NFS, S3*, selezionare *Abilita NFS*.
+
... Confermare la lingua predefinita.
... Aggiungere interfacce di rete.
... Aggiornare le informazioni dell'account amministratore della VM di storage (opzionale).


** Per le VM di archiviazione esistenti: selezionare *Cluster > VM di archiviazione*, selezionare la VM di archiviazione, selezionare *Impostazioni* e quindi selezionare image:icon_gear.gif["Icona delle azioni"] o image:icon-edit-pencil-blue-outline.png["Icona Edit (Modifica)"] sotto *NFS* e selezionare *Abilita NFS*.
+
... Abilitare o disabilitare versioni specifiche del protocollo NFS e opzioni secondo necessità.
+

NOTE: Il monitoraggio FPolicy degli eventi di accesso ai file NFSv4.2 non è supportato. Se si configura il monitoraggio FPolicy, è necessario disabilitare NFSv4.2 sulla SVM. Per riferimento, vedere link:nfs-admin/enable-disable-nfs-task.html["Abilitare o disabilitare NFS per SVM ONTAP"].





. Aprire il criterio di esportazione del volume root della VM di storage:
+
.. Seleziona *Archiviazione > Volumi*, seleziona il volume root della storage VM (che per impostazione predefinita è _volume-name_ _root), e quindi seleziona la policy visualizzata in *Export Policy*.
.. Seleziona *Aggiungi* per aggiungere una regola.
+
*** Specifica del client = `0.0.0.0/0`
*** Access Protocol (protocolli di accesso) = NFS
*** Dettagli di accesso = UNIX di sola lettura




. Configurare DNS per la risoluzione dei nomi host: selezionare *Archiviazione > Macchine virtuali di archiviazione*, selezionare la macchina virtuale di archiviazione, selezionare *Impostazioni* e quindi selezionare image:icon_gear.gif["Icona delle azioni"] sotto *DNS*.
. Configurare i name service secondo necessità.
+
.. Selezionare *Archiviazione > Macchine virtuali di archiviazione*, selezionare la macchina virtuale di archiviazione, selezionare *Impostazioni* e quindi selezionare image:icon_gear.gif["Icona delle azioni"] LDAP o NIS.
.. Seleziona image:icon_pencil.gif["Icona Edit (Modifica)"] nel riquadro Name Services Switch per includere eventuali modifiche.


. Configurare Kerberos se necessario:
+
.. Seleziona *Archiviazione > Macchine virtuali di archiviazione*, seleziona la macchina virtuale di archiviazione e quindi seleziona *Impostazioni*.
.. Seleziona image:icon_arrow.gif["Icona a forma di freccia"] nel riquadro Kerberos e poi seleziona *Aggiungi*.
