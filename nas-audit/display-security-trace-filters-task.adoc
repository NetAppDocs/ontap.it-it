= Visualizza informazioni sui filtri di tracciamento della sicurezza nelle SVM ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Visualizza i dettagli del filtro di traccia di sicurezza sulla tua storage virtual machine (SVM) per identificare gli eventi di accesso che ogni filtro traccia.

[role="tabbed-block"]
====
.System Manager
--
A partire da ONTAP 9.6, puoi utilizzare System Manager per tracciare l'accesso ai file sull'SVM.

.Fasi
. Aprire la dashboard di System Manager.
. In *Cluster*, seleziona *Storage VMs*.
. Selezionare image:icon_kabob.gif["Icona delle opzioni di menu"] per l'SVM applicabile.
. Seleziona *Trace file access*.
. Impostare il tipo di protocollo di accesso ai file su *SMB/CIFS* o *NFS*.
. Inserisci il *Nome utente*.
. Inserisci l'*indirizzo IP del client*.
. Facoltativamente, seleziona una o più delle seguenti:
+
.. Seleziona *Traccia percorso specifico* per specificare il file o il percorso del file da tracciare.
.. Selezionare *Mostra solo voci con accesso negato* per limitare la traccia agli eventi di accesso negato.


. Seleziona *Avvia tracciamento*.


--
.CLI
--
Utilizzare la CLI per tracciare l'accesso ai file sull'SVM.

.Fase
. Visualizzare le informazioni relative alle voci del filtro di traccia di protezione utilizzando `vserver security trace filter show` comando.
+
Ulteriori informazioni su `vserver security trace filter show` nella link:https://docs.netapp.com/us-en/ontap-cli/vserver-security-trace-filter-show.html["Riferimento al comando ONTAP"^].



.Esempi
Il seguente comando visualizza informazioni su tutti i filtri di traccia di sicurezza su SVM vs1:

[listing]
----
cluster1::> vserver security trace filter show -vserver vs1
Vserver  Index   Client-IP            Path            Trace-Allow  Windows-Name
-------- -----  -----------  ----------------------   -----------  -------------
vs1        1    -            /dir1/dir2/file.txt          yes      -
vs1        2    -            /dir3/dir4/                  no       mydomain\joe
----
